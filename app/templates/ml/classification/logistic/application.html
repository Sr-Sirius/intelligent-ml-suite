Logistic Regression - Application

Problem Description

This application aims to predict whether a transaction is fraudulent or legitimate using a classification model.

Dataset Explanation

The dataset contains transaction-related information such as amount, frequency, and other behavioral patterns. It is structured to allow the model to learn patterns associated with fraudulent activities.

Input Variables and Target Variable

  • Input variables: transaction amount, frequency, time
  • Target variable: fraud (1) or not fraud (0)

Data Preparation

The dataset is cleaned and preprocessed by removing missing values, normalizing numerical data, and splitting the data into training and testing sets.

Model Training Process

A Logistic Regression model is trained using labeled data. The model learns relationships between input features and the target variable to make accurate predictions.
