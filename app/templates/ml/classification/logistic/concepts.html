Logistic Regression - Basic Concepts

What is Logistic Regression?

Logistic Regression is a supervised machine learning algorithm used for solving classification problems, particularly when the output variable is categorical. Despite its name, it is not used for regression tasks but for predicting the probability that an input belongs to a specific class.

The model applies a mathematical function called the sigmoid function, which transforms any real-valued number into a value between 0 and 1. This allows the model to interpret outputs as probabilities, making it suitable for decision-making scenarios.

Logistic Regression is widely used due to its simplicity, interpretability, and effectiveness in binary classification problems.

Regression vs Classification

In machine learning, it is important to distinguish between regression and classification tasks.

Regression models are used to predict continuous numerical values, such as predicting house prices or temperature. In contrast, classification models are used to predict discrete categories or labels, such as whether an email is spam or not.

Logistic Regression belongs to the classification category because its objective is to assign inputs to predefined classes rather than predicting continuous outputs.

Binary Classification

Binary classification refers to problems where there are only two possible outcomes. These outcomes are typically represented as 0 and 1.

For example:

  • 0 (e.g., No Fraud)
  • 1 (e.g., Fraud)

Logistic Regression is specifically designed to handle binary classification by estimating the probability that a given input belongs to one of these two classes. Based on this probability, the model assigns the final class label.

Target Variable / Class Label

The target variable (also known as the class label) is the output that the model is trying to predict.

In classification problems, the target variable represents categories rather than numerical values. For example, in a fraud detection system, the target variable could be:

  • Fraud (1)
  • Not Fraud (0)

The model learns patterns from input data to accurately predict this target variable.

Features / Predictors

Features, also called predictors, are the input variables used by the model to make predictions.

These variables provide the information necessary for the model to identify patterns and relationships within the data. Features can include numerical values (such as age or income) or categorical values (such as location or product type).

The quality and relevance of features directly influence the performance of the model.

Probability Estimation

One of the key characteristics of Logistic Regression is that it outputs a probability value between 0 and 1.

This probability represents how likely it is that a given input belongs to a particular class. For example, a prediction of 0.85 indicates an 85% probability that the input belongs to class 1.

This probabilistic approach allows for flexible decision-making and interpretation of results.

Decision Boundary

The decision boundary is the point or line that separates different classes in the feature space.

In Logistic Regression, this boundary is determined based on the predicted probabilities. It defines the threshold at which the model switches from predicting one class to another.

For example, if the decision boundary is set at 0.5, values above this threshold are classified as one class, while values below are classified as another.

Classification Threshold

The classification threshold is the value used to convert predicted probabilities into class labels.

Typically, this threshold is set at 0.5:

  • If probability more or equalt to 0.5 → Class 1
  • If probability less than 0.5 → Class 0

However, this threshold can be adjusted depending on the problem. For example, in fraud detection, a lower threshold may be used to reduce the risk of missing fraudulent transactions.

Real-World Example

Logistic Regression is widely used in real-world applications where binary decisions are required.

Examples include:

  • Fraud detection in banking systems
  • Medical diagnosis (disease vs no disease)
  • Spam email filtering
  • Credit approval systems

Its ability to provide both predictions and probabilities makes it a powerful and interpretable tool in many industries.
